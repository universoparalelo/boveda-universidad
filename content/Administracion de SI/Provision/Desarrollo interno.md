Cuando conviene?
- Si la empresa tiene un area de sistemas.
- Si tiene recursos para desarrollar, tanto empleados con conocimientos como recursos informaticos como servidores, computadores, etc.
- Si la solucion requiere especificidad que no se encuentra en el mercado ni en una casa de software, es decir, si ningun fabricante puede ofrecer la realizacion del mismo.
- Si se puede permitir los costos y el tiempo no es poco o no es urgente tener el software.
- Se recomienda cuando es un sistema comun, no hay casas de software interesadas y tiene un impacto estrategico.
